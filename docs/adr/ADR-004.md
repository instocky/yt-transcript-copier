# ADR-0004 — Structured transcript segments + optional timestamps

## Status

Proposed

## Context

Current pipeline (v1.3) operates on plain text transcripts:

- timestamps are removed
- structure is limited to paragraphs

This limits:

- navigation
- future RAG use cases
- timestamp-based features

## Decision

Introduce segment-based transcript representation:

{
text: string,
start: number
}

Rendering layer becomes configurable:

- plain text (default)
- text with timestamps

## Consequences

- enables timestamps, navigation, indexing
- prepares pipeline for future features

* requires refactoring extraction layer
